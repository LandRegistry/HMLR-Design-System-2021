Why was this component needed?

The Portal Refresh team working on View My Applications are introducing a new update that allows users to see applications lodged by anyone in their organisation. This means that the user potentially has access to hundreds of different user's applications.

To allow the user to view all of the applications for a particular individual, a dropdown menu was introduced. This could mean potentially scrolling a list of hundreds of colleagues, by implementing search functionality within the dropdown menu itself, the user is able to locate a colleague quickly by typing the start of their name. As the user types, the search starts filtering the names in the dropdown menu.

When to use

This component should be used when the dropdown menu contains more than 20 options and no other standard GDS solution (check boxes, radio buttons) create a suitable user experience.

How to use

Dolor sit amet, consetetur sadipscing elitr, sed diam nonumy eirmo.

DO

When the dropdown menu has more than 20 options to select from.

DON'T

When the dropdown menu has just a few items in it.

Anatomy

  1. Form field: Uses same styling as GDS
  2. Icon: Use an icon to convey it's a dropdown menu
  3. Search: The form field becomes searchable when clicked on
  4. Highlighted: Selected item is more noticeable
  5. Scrollable: Exapnded list is scrollable if required
