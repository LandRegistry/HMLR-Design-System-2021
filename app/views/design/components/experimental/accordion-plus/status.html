Table inside accordion

This experimental component has been designed to allow users to easily identify and reveal the content relevant to them without having to look through unnecessary information.

This timeline shows the extent to which this experimental component has been tested, validated and approved.

User need identified

Design

02/09/2021

As part of a "grouping of services" workstream, Andy Carr and JRez identified the need for users to be able to interact with lots of data in a way that wasn't overwhelming. The standard GDS accordion was the starting point for this component, but it lacked some of the desired functionality e.g. interactive checkboxes
